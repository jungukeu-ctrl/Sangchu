## 3. index.html — 판단 가이드

### 구조
- **탭 3개**: 판단 트리 / A·B·C / 되돌림 계산기
- **판단 트리**: 3-step 플로우 (비중 → 평단대비 → 흐름) → 결과 카드
- **되돌림 계산기**: 고점·저점 입력 → 30%/50%/75% 자동 계산

### CSS 변수 (반드시 준수)
```css
--bg: #080808        /* 최외곽 배경 */
--bg-2: #111111      /* 카드 배경 */
--bg-3: #181818      /* 호버/선택 배경 */
--bg-4: #202020      /* 인풋/버튼 배경 */
--text-1: #f0ede8    /* 주 텍스트 */
--text-2: #888       /* 보조 텍스트 */
--text-3: #444       /* 비활성 텍스트 */
--red: #e24b4a       /* 매수/상승 */
--blue: #5b9bd5      /* 매도/하락 */
--amber: #d4a017     /* 경고/자동매도 */
--green: #4caf7d     /* 수익 */
--purple: #8b7fd4    /* 브랜드 포인트 */
```

### 핵심 로직
- `MAP` 객체: `"${weight},${priceZone},${flow}"` 키 → 판단 결과 (36가지 조합)
- `state`: `{ w, p, f }` — 각 스텝 선택값
- `calcDiff()`: 평단/현재가 → 수익/보합/손실권 분류
- `calcRetrace()`: (고점-저점) × 비율 + 저점

## 4. backtest.html — 상추훈련 백테스트

### 데이터 포맷 (JSON 입력)
```json
[
  { "d": "2024/01/02", "o": 4150, "h": 4320, "l": 4100, "c": 4280 },
  ...
]
```
- `d`: 날짜 (YYYY/MM/DD)
- `o/h/l/c`: 시가/고가/저가/종가
- 생성 도구: `naver_stock_fetcher.py` (외부 스크립트, 저장소에 없음)

### 상태 객체 (`st`)
```js
{
  cash: 1000000,   // 보유 현금
  shares: 0,       // 보유 주식 수
  avg: 0,          // 평단가
  real: 0,         // 실현손익
  tBuy: 0,         // 당일 매수 총액 (한도 계산용)
  tSell: 0,        // 당일 매도 총액 (한도 계산용)
  autoTh: 0,       // 자동매도 달성 임계값 (%)
  log: [],         // 거래 내역
  cur: -1          // 현재 날짜 인덱스
}
```

### 상추훈련 규칙 (코드에 하드코딩)
| 규칙 | 값 | 위치 |
|---|---|---|
| 자본금 | `CAP = 1,000,000` | 상수 |
| 일 최대 매수 | `CAP * 0.04` (4%) | `doBuy()`, `doNext()` |
| 1회 매수/매도 단위 | `CAP * 0.02` (2%) | `doBuy()`, `doSell()` |
| 수익 자동매도 | 평단 대비 +1% 구간마다 1% 자동 매도 | `clickDay()`, `doNext()` |
| 이월 불가 | `tBuy`는 `doNext()` 호출 시 리셋 | `doNext()` |

### 차트 (Canvas 2D)
```
Canvas 크기: CW × 220px (CW는 컨테이너 너비에 동적 맞춤)
padding: { l:8, r:60, t:16, b:24 }
캔들 폭: Math.max(1.5, pw/WIN - 1.2)
WIN: 표시 캔들 수 (기본 80, 줌 슬라이더로 20~600 조정)
sOff: 스크롤 오프셋 (왼쪽 시작 인덱스)
```

### 렌더링 레이어 순서 (draw 함수)
1. 그리드 라인 + 우측 가격 라벨
2. 평단가 점선 (황색, `st.avg > 0`일 때)
3. 캔들 (위꼬리 → 몸통 → 현재 위치 마커)
4. **거래 마커** (B/S 아이콘 — 저가 아래/고가 위)
5. 날짜 라벨 (하단)

### 거래 마커 로직
```js
// 날짜별 매수·매도 횟수 집계
const dateMap = {};
st.log.forEach(t => {
  if(!dateMap[t.day]) dateMap[t.day] = {b:0, s:0};
  if(t.type === 'buy') dateMap[t.day].b++;
  else dateMap[t.day].s++;
});
// B: 저가 아래 빨간 박스 + 꼬리
// S: 고가 위 파란 박스 + 꼬리
// 동일 날 복수 거래 시 숫자로 표시 (2, 3, ...)
```

### 컨트롤
| 버튼/슬라이더 | 동작 |
|---|---|
| 매수 | 선택일 시가 기준 2% 매수 |
| 매도 | 선택일 종가 기준 2% 매도 |
| 다음날 → | 자동 매도 + 자동 매수 실행 후 날짜 전진 |
| 초기화 | `st` 전체 리셋 |
| 스크롤 슬라이더 | `sOff` 조정 |
| 줌 슬라이더 | `WIN` 조정 (20~600일) |

## 6. 매매 철학 — 코드 수정 시 반드시 숙지

> 상추훈련은 단순 주식 앱이 아니다. **빌드업 매매 철학을 코드로 구현한 교육 도구**다.

### 핵심 원칙 (위반하는 코드 작성 금지)
1. **일 4% 한도** — 이월 불가, 매도 시 한도 복구
2. **분할 매수** — 한방 매수 없음, 1~2% 단위
3. **3가지 계산 우선** — 평단 변화 / 비중 변화 / 기준비중 대비
4. **덩어리 유지** — 전량 매도 후 재진입 흐름도 고려

### 1-2-3 박스 이론
- **1번 박스**: 워밍업, 준비 단계 → 소량 진입
- **2번 박스**: 본격 상승 → 드리블(비중 줄이며 수익)
- **3번 박스**: 눌림 → 훅 확인 후 판단

### 수정 전 반드시 확인

**backtest.html 수정 시**
```
[ ] draw() 함수 — 레이어 순서 영향 여부
[ ] exec() 함수 — st 객체 변형 → render(), draw() 연쇄 영향
[ ] clickDay() / doNext() — 자동매도 로직 중복 여부 확인
[ ] WIN / sOff 변경 — scrl.max 재계산 필요 여부
[ ] CAP 상수 — 모든 % 계산에 영향 (grep CAP 선행)
```

**index.html 수정 시**
```
[ ] MAP 객체 — 36개 키 모두 커버되는지 확인 (0~3, 0~2, 0~2 조합)
[ ] state 객체 — w/p/f 세 값이 항상 null 아닌 상태에서만 결과 출력
[ ] CSS 변수 — var(--xxx) 참조 누락 여부
[ ] 탭 전환 — panel.active / tab.active 클래스 동기화
```

**양 파일 공통**
